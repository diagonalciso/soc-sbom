📦 SBOMguard User Manual

SBOM vulnerability tracker and CVE monitor

Overview

SBOMguard monitors Software Bill of Materials (SBOM) files for known vulnerabilities. Upload SBOM JSON, scan against NVD, KEV, EPSS, and OSV databases, track remediation.

Getting Started

  1. Navigate to http://localhost:8082
  2. Upload SBOM file (JSON format) via web UI
  3. View vulnerability scan results
  4. Filter by CVSS threshold, severity, status
  5. Track remediation progress

Uploading SBOMs

Click "Upload SBOM" and select JSON file. Supported formats:

  • CYCLONEDX (CycloneDX)
  • SPDX (SPDX 2.1+)
  • Generic JSON with package/version fields

Vulnerability Scanning

SBOMguard automatically scans uploads against:

  • NVD — National Vulnerability Database (all CVEs)
  • CISA KEV — Known Exploited Vulnerabilities (actively exploited)
  • EPSS — Exploit Prediction Scoring System (exploitation likelihood)
  • OSV — Open Source Vulnerabilities

Filtering & Sorting

  • CVSS Threshold: Show only vulnerabilities above score (0-10)
  • Severity: Critical, High, Medium, Low, Info
  • Status: Open, Remediated, Waived, In Progress
  • Sort: By CVSS, EPSS, date discovered, status

Remediation Tracking

Mark vulnerabilities as:

  • Open — Not yet addressed
  • In Progress — Patch pending/testing
  • Remediated — Fixed in latest version
  • Waived — Accepted risk

Export & Reporting

Export scan results as CSV or JSON for compliance reporting.

FAQ

Q: How often are CVE databases updated?
A: Daily. New vulnerabilities added to NVD/KEV within 24 hours.

Q: Can I upload multiple SBOMs?
A: Yes. Each SBOM tracked separately with history.

Q: What's EPSS?
A: Exploit Prediction Scoring System—likelihood exploit will be released (0-1 scale).
