# SBOMguard Administration Manual

SBOMguard is a **SBOM vulnerability tracker**. An SBOM (Software Bill of Materials) is a list of all libraries/dependencies in an application. SBOMguard checks that list against vulnerability databases (NVD, KEV, OSV) to find dangerous packages.

**Use case:** "We deployed app v2.0. What vulnerabilities does it have?"

---

## Installation

### Requirements

- Python 3.8+
- 500 MB disk (database grows slowly)
- Internet access (downloads vulnerability feeds)

### Setup

```bash
cd ~/claude/SBOMguard
cp .env.example .env
nano .env

# Edit:
SBOMGUARD_PORT=8082
CVE_FEED=nist              # or: nist, osv, keycert
CVSS_THRESHOLD=7.0         # Alert on CVSS >= 7.0
```

### Run

```bash
# Direct
python3 app.py

# Or systemd
sudo cp sbomguard.service /etc/systemd/system/
sudo systemctl enable --now sbomguard
sudo journalctl -u sbomguard -f
```

## Configuration

```env
SBOMGUARD_PORT=8082          # Web UI port
CVE_FEED=nist               # Vulnerability source (nist/osv/keycert)
CVSS_THRESHOLD=7.0          # Min severity to alert on
MAX_RESULTS=1000            # Max vulnerabilities to track
NVD_API_KEY=                # Optional: faster NVD fetches
CACHE_TTL=3600              # Cache vulnerability data for N seconds
```

## Uploading SBOMs

### Format Support

- **CycloneDX** (JSON/XML) — industry standard
- **SPDX** (JSON/XML/TagValue)
- **Syft** (JSON) — generated by Syft tool

### Via Web UI

1. Open `http://localhost:8082`
2. Click "Upload SBOM"
3. Select file
4. See vulnerabilities immediately

### Via API

```bash
curl -X POST http://localhost:8082/api/sbom/upload \
  -F "file=@app-sbom.json" \
  -F "app_name=MyApp" \
  -F "version=2.0"
```

## Database

SQLite at `~/claude/SBOMguard/sbomguard.db`

```bash
# Inspect
sqlite3 sbomguard.db
SELECT COUNT(*) FROM vulnerabilities;
SELECT * FROM sboms;

# Backup
cp sbomguard.db sbomguard.db.backup

# Clean old SBOMs (>90 days)
sqlite3 sbomguard.db "DELETE FROM sboms WHERE created_at < datetime('now','-90 days')"
```

## Vulnerability Feeds

Fetched automatically daily:

| Feed | Source | Updates | Coverage |
|------|--------|---------|----------|
| NVD | NIST | Daily | ~250k CVEs |
| KEV | CISA | Daily | 1000+ exploited CVEs |
| OSV | Google | Daily | Open source focus |

No configuration needed — runs automatically.

## Troubleshooting

**Port in use:**
```bash
lsof -i :8082 && kill -9 <PID>
```

**No vulnerabilities found:**
- Check SBOM format (CycloneDX/SPDX are supported)
- Verify feeds loaded: `sqlite3 sbomguard.db "SELECT COUNT(*) FROM cves"`

**Slow searches:**
- Feeds might be updating. Wait 5 min.
- Check disk space: `df -h`

## Collection Agents

Optional: Linux/Windows agents in `sbomguard/agents/` automatically generate SBOMs from installed packages.

```bash
# Linux agent
./agents/linux-sbom-collector.sh > app-sbom.json

# Windows agent (PowerShell)
./agents/windows-sbom-collector.ps1 > app-sbom.json

# Upload to SBOMguard
curl -X POST http://sbomguard:8082/api/sbom/upload -F "file=@app-sbom.json"
```

Configure cron/Task Scheduler for periodic collection.
